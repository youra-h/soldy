# Советы по работе с Copilot

На основе анализа сессий от 17.07.2026.

---

### Совет 1: Используй `/compact` для компрессии контекста

Сессия из 35 turn'ов — идеальный кандидат. После каждых **8–12 turn'ов** или после крупного архитектурного решения используй `/compact`. Это сжимает историю в краткое резюме, и следующие шаги не тащат за собой 20 turn'ов обсуждений неправильных подходов. Особенно актуально, когда ты говоришь «Нет. стоп. Я это все удаляю» — после отката стоит сразу сделать `/compact`, чтобы забыть устаревшие варианты.

### Совет 2: Создай `.github/copilot-instructions.md` с архитектурными принципами

Ты тратишь много turn'ов на объяснение базовой архитектуры: схема как контейнер, слои core → schema → plugins, разделение `props/events/readonly`. Запиши это в `.github/copilot-instructions.md` (в корне проекта). Пример содержания:

```
Проект soldy — монорепа. Ключевые пакеты: schema, core, plugins, ui/*.

Архитектура схемы:
- Schema — самодостаточный контейнер: props, events, readonly, Ctor.
- entitySchema — базовая (ctrl, plugins).
- componentSchema — расширяет entitySchema (rendered, visible, present).
- Плагины расширяют схему через mergePluginSchemas().
- createAdapter, createBundle, createRefs — runtime-функции.

Правила: не добавляй framework-логику в core, используй #sym: для ссылок на код.
```

Это сократит 3–4 turn'а контекста на каждую новую сессию.

### Совет 3: Начинай новую сессию после смены направления

Когда ты говоришь «Слушай, мне текущий подход сейчас совершенно не нравится» — это сигнал начать **новую сессию**. Вместо того чтобы дописывать в конец 30-turn'овой сессии, открой новый чат, сделай `/compact` в старом для истории, и объясни новый подход с чистым контекстом. Модель не будет отвлекаться на предыдущие ошибочные решения.

### Совет 4: Используй подзадачи (Explore-агент) для исследовательских вопросов

Моменты вроде «сложная задача для понимания. Давай ее обсуждать» или «чего нам сейчас не хватает в этой схеме?» — исследовательские обсуждения, которые можно выносить в подагента **Explore**. Он проведёт ресерч в изолированном контексте, а вернёт только суть. Это не будет раздувать основной контекст сессии.

### Совет 5: Используй `#file:` вместо `#sym:` для точных ссылок

`#sym:` хорошо, но на файлы, которые ты правишь несколько раз подряд, лучше ссылаться через `#file:packages/schema/runtime/createAdapter.ts` — это даёт модели полный контекст файла, и ей не нужно переспрашивать или додумывать.
